CS50 Intro Lecture Today, we will review functions. 1) A block of code that together perform specific task (or function). 2) The block sits idle until called by name (like tools sitting in your toolbox until needed). 3) The function may have parameters to make it adjustable. or it may be single purpose... 4) Functions allow for abstraction or detail hiding. Few of us care how a wrench is made. We just want to use it without any hassles. Functions may have some restrictions or limits on their use. (Continuing the tool analogies, think chainsaws). 5) We can reuse functions whenever needed without having to rewrite (reforge) our tool. 6) We can share our function with others. 7) An initial version may be inefficient/limited, but allows for use in main program. 8) Functions can be revised and if interface is unchanged, easily plugged into main program. 9) All programming languages provide functions (also called subroutines or methods). 10) Some functions are part of the standard release. Others are part of add-on libraries. 11) Additionally, programmers (like you!!!) can create your own function. 12) For large projects, team members can agree on function interfaces and then design in parallel. 13) A function in SCRATCH... polygon
